## Подключение клиентов

Нужно сгенерировать конфигурационный файл. Т.к. я использовал образ докера `kylemanna/docker-openvpn`, вызываю команду на машине с контейнером сервера, предварительно заменив `${client_name}` на имя клиента:

```bash
docker-compose run --rm openvpn easyrsa build-client-full ${client_name} nopass
```

Произойдет генерация сертификатов. Теперь нужно экспортировать сертификаты в файл:

```bash
docker-compose run --rm openvpn ovpn_getclient {client_name} > certificate.ovpn
```

и закидывать на машину клиента.

Во всех инструкциях и видео, для подключения клиентов по статическим ip, не было объяснений какие ip доступны. В итоге нашел информацию [про таблицу для присвоения ip адресов](https://forums.openvpn.net/viewtopic.php?t=8406).

Это прописано в файле конфигурации, ip и маска может отличаться:

```
server 10.3.0.16 255.255.255.240
```

Из [документации OpenVPN](https://openvpn.net/community-resources/how-to/#policy) — последний октет в IP адресе каждой пары endpoint должен быть из следующего набора:

```
[  1,  2] [  5,  6] [  9, 10] [ 13, 14] [ 17, 18]
[ 21, 22] [ 25, 26] [ 29, 30] [ 33, 34] [ 37, 38]
...
```

### Клиент на Linux

Полезный мануал [тут](https://hackware.ru/?p=5369). Там описано сработавшее решение и добавление в автозагрузку.

Закидываем файл `*.ovpn` по пути `/etc/openvpn/client/client.conf`

```bash
# запуск
sudo systemctl start openvpn-client@client.service
# добавление в автозагрузку
sudo systemctl enable openvpn-client@client.service
# проверка
sudo systemctl status openvpn-client@client.service
```

#### Решение проблемы работы клиента и Docker

Столкнулся с тем что контейнеры в docker не хотели запускаться, а запущенные остановились.
На другом сервере, во время попытки использования docker-compose, была ошибка:

```
ERROR: could not find an available, non-overlapping IPv4 address pool among the defaults to assign to the network
```

Связал это с запущенным клиентом OpenVPN. Нашел [решение](https://stackoverflow.com/questions/45692255/how-make-openvpn-work-with-docker).

Нужно создать файл `/etc/openvpn/fix-routes.sh`:

```bash
#!/bin/sh

echo "Adding default route to $route_vpn_gateway with /0 mask..."
ip route add default via $route_vpn_gateway

echo "Removing /1 routes..."
ip route del 0.0.0.0/1 via $route_vpn_gateway
ip route del 128.0.0.0/1 via $route_vpn_gateway
```

Сделать его исполняемым и изменить владельца:

```bash
chmod o+x /etc/openvpn/fix-routes.sh
chown root:root /etc/openvpn/fix-routes.sh
```

Добавить строки в конфигурационный файл клиента:

```
script-security 2
route-up  /etc/openvpn/fix-routes.sh
```

#### Как исключить проброс исходящего трафика через сеть openvpn

В файле конфигурации клиента `/etc/openvpn/client/client.conf` закомментировать/удалить строку:

```
redirect-gateway def1
```

и добавить следующие:

```
pull-filter ignore redirect-gateway
route 0.0.0.0 192.0.0.0 net_gateway
route 128.0.0.0 192.0.0.0 net_gateway
```

[Похожая информация тут](https://community.openvpn.net/openvpn/wiki/IgnoreRedirectGateway)
